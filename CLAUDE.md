## What is Ferrite

Ferrite is a native macOS .NET decompiler. A Rust backend parses PE/CLR metadata from .NET assemblies (.dll/.exe); a SwiftUI frontend displays a browsable tree of namespaces, types, and members. Currently in Phase 1 (assembly browser). Later phases add IL disassembly and C# decompilation.

## Architecture

```
SwiftUI app  ──UniFFI FFI──▸  Rust static library
(Sources/)                    (rust/)
```

- **ferrite-pe** — Core .NET PE parsing crate. Pipeline: PE headers (goblin) → CLR header → metadata root → stream headers → #~ table rows → #Strings heap → `Assembly` model.
- **ferrite-ffi** — UniFFI boundary crate (proc-macro mode, no UDL files). Compiles to `staticlib`. Exports `DecompilerSession` (Object), records (`AssemblyInfo`, `TypeInfo`, etc.), enums (`TypeKind`, `MemberKind`), and `FerriteError`.
- **FerriteFFI** — Thin C target (`Sources/FerriteFFI/`) containing only the generated header and modulemap so Swift can `import ferrite_ffiFFI`.
- **Ferrite** — SwiftUI app target. `DecompilerService` (@Observable) wraps the FFI session; views consume it via environment.

## Build Commands

```bash
# Full build: compile Rust + generate Swift bindings
make all

# Or step by step:
cd rust && cargo build --release          # build Rust crates
./scripts/build-rust.sh                    # build + uniffi-bindgen + copy artifacts

# Regenerate Xcode project from project.yml (after changing targets/settings)
xcodegen generate

# Build Swift (CLI)
swift build

# Build via Xcode
xcodebuild -project Ferrite.xcodeproj -scheme Ferrite build

# Run the app
open Ferrite.xcodeproj   # then Cmd+R in Xcode
```

## Tests

```bash
cd rust && cargo test           # Rust unit tests (in ferrite-pe)
cd rust && cargo test -p ferrite-pe  # just the PE crate
```

There are no Swift tests yet. Rust tests live in `rust/ferrite-pe/src/assembly.rs`.

## Build Workflow After Rust Changes

Any change to Rust code requires regenerating the static library and Swift bindings before the Swift side will see it:

1. `make all` (or `./Scripts/build-rust.sh`)
2. Rebuild in Xcode (Cmd+B)

The script auto-detects arm64 vs x86_64, runs `cargo build --release --target <arch>`, runs `uniffi-bindgen generate`, and copies `libferrite_ffi.a` + `ferrite_ffiFFI.h` to the right places.

## FFI Conventions

- UniFFI proc-macro mode: types annotated with `#[uniffi::Record]`, `#[uniffi::Enum]`, `#[uniffi::Object]`, `#[uniffi::Error]`.
- Rust `snake_case` fields become Swift `camelCase` in generated bindings (e.g. `full_name` → `fullName`, `nested_types` → `nestedTypes`).
- Swift keywords in enum variants get backtick-escaped: `.class` → `` .`class` ``, `.struct` → `` .`struct` ``, `.enum` → `` .`enum` ``.
- `DecompilerSession` is an `Arc`-wrapped Object; constructor is `DecompilerSession.open(path:)` in Swift.

## .NET Metadata Parsing Notes

- ECMA-335 spec governs all metadata structures.
- TypeKind detection: interface flag (0x20) → Interface; extends System.Enum → Enum; extends System.ValueType → Struct; extends System.MulticastDelegate → Delegate; else → Class.
- Table rows are 1-indexed. Tokens encode table ID in high byte (TypeDef=0x02, MethodDef=0x06, Field=0x04).
- Nested types use the NestedClass table to map child→parent; they're removed from the top-level list and attached to their enclosing type.
- Row 1 of TypeDef is always the `<Module>` pseudo-type and is filtered out.
